LEVANTAR SERVIDOR --- npm run dev

------------------------------------------------------------------------------------------------------------------------------------

FETCH --- es una función nativa de JavaScript que sirve para hacer peticiones HTTP (por ejemplo, pedir datos a una API).

respuesta.ok → true si el status es 200–299 (es decir, éxito)

respuesta.status → el código HTTP (200, 404, 500, etc.)

respuesta.json() → un método que lee el cuerpo y lo convierte a un objeto JS.

-------------------------------------------------------------------------------------------------------------------------------------

AWAIT --- se usa solo dentro de funciones async y su función es esperar a que una promesa se resuelva antes de continuar.

-------------------------------------------------------------------------------------------------------------------------------------

PAYLOAD --- Esto hace que se carguen los datos importantes de un json y hacen que se muestren diferentes como un pretty de mongo

--------------------------------------------------------------------------------------------------------------------------------------

CONTROLLERS --- Se colocan las funciones que contienen la lógica real de la aplicación.
                Los controladores dicen qué hacer cuando ese endpoint es llamado.
                Esta la lógica principal de cada endpoint.
--------------------------------------------------------------------------------------------------------------------------------------

MODELS ---  Contiene todas las funciones que permiten interactuar con la base de datos.
            Aqui se hacen todas las consultas SQL de la Pagina

--------------------------------------------------------------------------------------------------------------------------------------

API --- Una api permite que dos programas se comuniquen entre si

API REST --- La API REST es el puente, usando endpoints como puertas de acceso (GET / POST / PUT / DELETE)

--------------------------------------------------------------------------------------------------------------------------------------

ENDPOINT --- Es una dirección (URL) donde puedes pedir o enviar información.

--------------------------------------------------------------------------------------------------------------------------------------

RENDERIZAR ---  Renderizar significa convertir algo en una salida visible.

                En una web, es el proceso de:
                ➡️ Transformar código (plantillas, datos, lógica) en una página que el usuario puede ver
                ➡️ Mostrar textos, tablas, imágenes, botones, formularios, etc.

--------------------------------------------------------------------------------------------------------------------------------------

EJS ---  Son plantillas HTML
         Con pedacitos de código JS dentro
         Que se procesan en el servidor para producir HTML final
         Tambien sirve para reutilizar componentes 

--------------------------------------------------------------------------------------------------------------------------------------

ROUTES ---  La carpeta routes sirve para organizar las rutas de tu servidor, es decir, 
            los caminos/endpoints que tu backend escucha y responde.

            index.js arranca el servidor
            routes/archivo.js maneja qué pasa cuando alguien visita una ruta

--------------------------------------------------------------------------------------------------------------------------------------

PUBLIC --- Public sirve para alojar archivos estáticos que el navegador puede acceder directamente.

ARCHIVOS ESTATICOS --- Son archivos que no necesitan lógica del servidor para generarse, sino que solo se envían tal cual
